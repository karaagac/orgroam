Api gataway provides several important functions:

- Authentication
- Security policy enforcement
- Load balancing and circuit breaking (Circuit breaking is a pattern that stops sending requests to a service that is failing, so it doesn't become even more overloaded.)
- Protocol Translation and service discovery
- Monitoring, Logging, analytics and billing
- Caching

Step 1: Client send a request to api gataway. request is typically HTTP based. It can be REST, GRAPHQL or some other high level abstraction.

Step 2: Api Gataway validates the HTTP request.

Step 3: Api Gataway checks callers ip address and other HTTP Headers against its allow list and deny list.

Step 4: Api Gataway passes request to identity provider for authentication and authorization. Api Gataway receives an authenticated session back from the provider with the scope of what the request is allowed to do.

Step 5: High level rate-limit check is applied against the authenticated session. If its over the limit request is rejected at this point.

Step 6 and 7: With the help of a service discovery component, the AP gataway locates the appropriate backend service to handle the request by path matching.

Step 8: Api Gataway transforms the request into the appropriate protocol and send transformed request to the backend service.


When response comes back from the backend service, the API gataway transforms the response back to the public-facing protocol and returns the response to the client.

A proper API gataway also provides other critical services. For example: an API gataway should track errors and provides circuit breaking functionality to protect the services from overloading.

An API gataway should also provide logging, monitoring and analitics services for operational observability.

An API gataway is a critical piece of the infrastructure. It should be deployed to multiple regions to improve availability.

For many cloud provider offerings, the API gataway is deployed across the world close to the clients.
