* Playwright

** What are fixtures
Fixtures are pre-configured objects/resources that Playwright automatically creates and injects into your test function.

#+BEGIN_SRC javascript
test('example', async ({ page, browser, context }) => {
#+END_SRC

Here:

- page → a browser tab
- browser → the browser instance
- context → an isolated browser session

These are fixtures.

=Think of fixtures as:=

“Things Playwright prepares for your test so you don’t have to manually create/setup them every time.”

Without fixtures, you would write more setup code manually:

#+BEGIN_SRC javascript
const browser = await chromium.launch();
const context = await browser.newContext();
const page = await context.newPage();
#+END_SRC

With fixtures, Playwright can do much of this automatically:

#+BEGIN_SRC javascript
test('example', async ({ page }) => {
    await page.goto('https://example.com');
});
#+END_SRC


=Why fixtures are useful=

Fixtures help with:

- setup/teardown automation
- reusable test state
- cleaner tests
- dependency injection
- test isolation
